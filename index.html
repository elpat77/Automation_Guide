Website Overview

This website was built as a place to keep information on how to to run automation using different programing languages.

It contains things such as installation, tools needed and scripts to use in order to run tests.

Important Steps to take before writting automation:

    Number one, it's all about deciding what features to automate. This requires having knowledge of the application under test and all of the available functionality.
    Number two, outline the scenarios that will be automated, and the steps of each test. Be aware of what the inputs and outputs will be, and the value each scenario implemented will provide.
    Number three, find the web elements that will be needed for testing. It's important to figure out what elements are going to be used in the tests upfront. This will make test writing much easier, faster, and smoother

This guide was created in order to use Selenium Webdriver with Ruby using Mac OS.

There are several methods to install Ruby, including package managers, which are the system-provided tools that manage software packages; installers, which install specific versions of Ruby; managers, which help switch between multiple Ruby installations, and from source code.

I recommend using the manager RVM, which stands for the Ruby Version Manager, it's widely used, and allows you to quickly and easily install Ruby.

    Using WebDriver

    The Webdriver API docs can be found on the following site:

    Webdriver API Docs

    WebDriver is the perfect option for those who have a need to test their application across multiple browsers and platforms. There is a lot of customization to use the WebDriver API, which makes it a very powerful testing framework. If you need to automate web tests, WebDriver is definitely the way to go.
    Selenium WebDriver works using client server communication. When a Selenium test is executed, a new session of the browser driver is created and a browser is launched.
    For each command in the test script, a request is sent to the WebDriver API. The WebDriver API interprets the request and the step is then executed in the browser driver, which acts as the server and just waits for the request to come in. Once each step is complete, the response is sent back to the WebDriver API and then back to the test. This process continues until all steps are complete.

    Sample Test

    We will be using the Ruby API documentation for WebDriver.
    Familiarize yourself with all the classes and methods available to perform actions on the browser.


    The following is a simple sample test using such classes and methods:

    Download Sample Code

    # 1) Requires the selenium-webdriver gem in order to use the webdriver API library
    require "selenium-webdriver"
    # 2) Using the Driver Class:
    # Set-up the driver, in this case we create a new instance of the Firefox driver.
    driver = Selenium::WebDriver.for :firefox
    # Then we use the same driver to navigate into the google URL
    driver.navigate.to "http://google.com"
    # 3) We now use the Search Context Class to find elements, which will find the first element matching the given arguments
    # There are many options available to find an HTML element, such as class name, CSS selector, ID, or tag name.
    # In this case, we are finding an element with the name 'q'
    element = driver.find_element(name: 'q')
    # 4) After finding an element, the next step is to perform some action on that element. This all happens within the Element Class.
    # In the element class, there are tons of options that can be used to drive action on elements.
    # In this case we are going to click.
    element.click
    # In this case we are going to type (send keys)
    element.send_keys "Hello WebDriver!"
    # In this case we are clicking and submitting the specified form element
    element.submit
    # Herfe we will close the driver session
    driver.quit
